Git Начало

Установка Git

Вы можете бесплатно скачать Git со следующего веб-сайта: https://www.git-scm.com/

Использование Git с командной строкой

Чтобы начать использовать Git, мы сначала откроем вашу командную оболочку.

Для Windows вы можете использовать Git bash, который входит в состав Git для Windows. Для Mac и Linux вы можете использовать встроенный терминал.

Первое, что нам нужно сделать, это проверить, правильно ли установлен Git:

Example

$ git --version

git version 2.30.2.windows.1

Если Git установлен, он должен показывать что-то вроде 'git version X.Y'

Конфигурация Git

Теперь дайте Git знать, кто вы такой. Это важно для систем контроля версий, поскольку каждая фиксация Git использует эту информацию:

Example

$ git config --global user.name "w3schools-test"

$ git config --global user.email "test@w3schools.com"

Измените имя пользователя и адрес электронной почты на свои собственные. Вероятно, вы также захотите использовать это при регистрации на GitHub позже.

Примечание: Используйте global чтобы задать имя пользователя и адрес электронной почты для каждой репозитории на вашем компьютере.

Если вы хотите установить имя пользователя / адрес электронной почты только для текущего репозитория, вы можете удалить global

Создание Git репозитория

Теперь давайте создадим новую папку для нашего проекта:

Example

$ mkdir myproject

$ myproject

mkdir Создает новую папку.

cd изменяет текущий рабочий каталог.

Теперь, когда мы находимся в правильном каталоге. Мы можем начать с инициализации Git!

Примечание: Если у вас уже есть папка / каталог, который вы хотели бы использовать для Git:

Перейдите к нему в командной строке или откройте его в проводнике, щелкните правой кнопкой мыши и выберите "Git Bash here"

Инициализация Git

Как только вы перейдете к нужной папке, вы можете инициализировать Git в этой папке:

Example

$ git init

Initialized empty Git repository in /Users/user/myproject/.git/

Вы только что создали свой первый репозиторий Git!

Примечание: Git теперь знает, что он должен следить за папкой, в которой вы его запустили.

Git создает скрытую папку для отслеживания изменений.
