Git New Files

Git добавление новых файлов

Вы только что создали свое первое локальное репозиторий Git. Но он пуст.

Итак, давайте добавим несколько файлов или создадим новый файл с помощью вашего любимого текстового редактора. Затем сохраните или переместите его в папку, которую вы только что создали.

Для этого примера я собираюсь использовать простой HTML-файл

И сохраню его в нашей новой папке как index.html .

Давайте вернемся к терминалу и перечислим файлы в нашем текущем рабочем каталоге:

Example

$ ls

index.html

ls выведет список файлов в каталоге. Мы можем видеть, что index.html есть там.

Затем мы проверяем статус Git и смотрим, является ли он частью нашего репозитория:

Example

$ git status

On branch master

No commits yet

Untracked files:

(use "git add ..." to include in what will be committed) index.html

nothing added to commit but untracked files present (use "git add" to track)

Теперь Git знает об этом файле, но не добавил его в наш репозиторий!

Файлы в вашей папке репозитория Git могут находиться в одном из 2 состояний:

  • Tracked - файлы, о которых Git знает и которые добавляются в репозиторий
  • Untracked - файлы, которые находятся в вашем рабочем каталоге, но не добавлены в репозиторий

Когда вы впервые добавляете файлы в пустой репозиторий, все они не отслеживаются. Чтобы заставить Git отслеживать их, вам нужно создать их или добавить в промежуточную среду.
