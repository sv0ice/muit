Git Commit

Git Commit

Поскольку мы закончили нашу работу, мы готовы перейти от stage к commit нашего репозитория.

Добавление коммитов позволяет отслеживать наш прогресс и изменения по мере работы. Git рассматривает каждую точку изменения фиксации или "точку сохранения". Это точка в проекте, к которой вы можете вернуться, если обнаружите ошибку или захотите внести изменения.

Когда мы коммитим, мы всегда должны включать сообщение.

Добавляя четкие сообщения к каждому коммиту, вам (и другим) будет легко увидеть, что изменилось и когда.

Example

$ git commit -m "First release of Hello World!"

[master (root-commit) 221ec6e] First release of Hello World!

3 files changed, 26 insertions(+)

create mode 100644 README.md

create mode 100644 bluestyle.css

create mode 100644 index.html

Команда commit выполняет снимок, а -m "message" добавляет сообщение.

Промежуточная среда была привязана к нашему репозиторию с сообщением: "Первый выпуск Hello World!"
