Введение в Git и Github

Что такое Git?

Git - популярная система контроля версий. Он был создан Линусом Торвальдсом в 2005 году и с тех пор поддерживается Хунио Хамано.

Он используется для:

  • Отслеживание изменений кода
  • Отслеживание того, кто внес изменения
  • Сотрудничество в области разработки проекта

Что делает git?

  • Управление проектами с помощью Repositories
  • Clone проект для работы с локальной копией
  • Контролирует и отслеживает изменения с помощью Staging and Committing
  • Branch and Merge чтобы обеспечить возможность работы над различными частями и версиями проекта
  • Pull последнюю версию проекта в локальную копию
  • Push локальные обновления основного проекта

Работа с Git

  • Инициализируйте Git в папке, сделав ее Repository
  • Git теперь создает скрытую папку, чтобы отслеживать изменения в этой папке
  • Когда файл изменяется, добавляется или удаляется, он считается modified
  • Вы выбираете измененные файлы, которые хотите сохранить Stage
  • The Staged files are Committed, which prompts Git to store a permanent snapshot of the files
  • Git позволяет вам просматривать полную историю каждого коммита.
  • Вы можете вернуться к любому предыдущему снимку.
  • Git не хранит отдельную копию каждого файла в каждом снимке, но отслеживает изменения, внесенные в каждый снимок!

Почему Git?

  • Более 70% разработчиков используют Git!
  • Разработчики могут работать вместе из любой точки мира.
  • Разработчики могут ознакомиться с полной историей проекта.
  • Разработчики могут вернуться к более ранним версиям проекта.

Что такое GitHub?

  • Git - это не то же самое, что GitHub.
  • GitHub создает инструменты, использующие Git.
  • GitHub является крупнейшим хранилищем исходного кода в мире и принадлежит Microsoft с 2018 года.
  • В этом руководстве мы сосредоточимся на использовании Git с GitHub.
