Git Staging Environment

Git Staging Environment

Одной из основных функций Git являются концепции промежуточной среды и фиксации.

Во время работы вы можете добавлять, редактировать и удалять файлы. Но всякий раз, когда вы достигаете важной вехи или завершаете часть работы, вам следует добавлять файлы в промежуточную среду.

Staged файлы - это файлы, которые готовы к commit в репозиторий, над которым вы работаете. Вскоре вы узнаете больше о фиксации.

На данный момент мы закончили работать с index.html . Таким образом, мы можем добавить его в промежуточную среду (staging env):

Example

$ git add index.html

Файл должен быть подготовлен. Давайте проверим статус:

Example

$ git status

On branch master

No commits yet

Changes to be committed:

(use "git rm --cached ..." to unstage)

new file: index.html

Теперь файл добавлен в промежуточную среду.
